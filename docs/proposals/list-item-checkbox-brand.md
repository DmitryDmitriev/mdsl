# List Item — расширение Left Side: Checkbox + Brand

**Статус:** proposal, ждёт ответа разработки. После апрува — правки в `list-item-spec.md` + компонент в Figma.

**Контекст:** запрос возник на задаче мультиселекта марок (PB-876), но проблема системная — не привязана к одному экрану. На row нужно **два** элемента слева одновременно: чекбокс выбора **и** логотип бренда.

**Текущее состояние компонента:** `List Item / Left Side` поддерживает ровно один `Type` на ряд (Checkbox **или** Avatar **или** Icon). Нет варианта «checkbox + что-то ещё».

## Предложение

Не делать отдельный List Item. Расширить существующий `List Item / Left Side`, добавив composite-варианты:

| Новый variant | Состав | Кейсы |
|---|---|---|
| `Type=Checkbox + Brand` | `[Checkbox 24]` + gap + `[Brand 40×40]` | Выбор марок (PB-876), выбор моделей с логотипом |
| `Type=Checkbox + Icon` *(на будущее)* | `[Checkbox 24]` + gap + `[Icon 24]` | Выбор категорий, опций с иконкой |

**Внутреннее устройство Left Side:**
- HORIZONTAL auto-layout
- `gap = spacing/2` (8) между checkbox и brand/icon
- Высота — по высоте самого крупного элемента (40 для Brand-варианта)

**Композиция всей row не меняется:** Left Side остаётся одним слотом с одной property `Type`. Для разработки — это всё ещё один `LeftSide`-композит, просто внутри две view'хи вместо одной.

## Почему не отдельный List Item

- Форк = двойная поддержка, расхождения токенов/стейтов со временем
- Сегодня — Brand. Завтра — `Checkbox + Icon`, `Radio + Avatar`, `Checkbox + Initials`. Через год — 8 параллельных листайтемов
- Правки самого List Item (новые стейты, изменения паддингов, новые токены) придётся применять везде вручную

2. Если есть конкретное «сложно» — что именно? Скорее всего речь про связку «checkbox state ↔ row click» в Compose. Это инженерная задача, не повод форкать компонент

## Что НЕ входит в компонент

- **Divider между rows** — НЕ часть List Item. Composition-уровень: дивайдер ставится снаружи между rows экрана, если нужен. По умолчанию — пустота 0, без дивайдера

# List Item — расширение Left Side: Checkbox + Avatar (для брендов)

**Статус:** proposal, ждёт ответа разработки. После апрува — правки в `list-item-spec.md` + компонент в Figma.

**Контекст:** запрос возник на задаче мультиселекта марок (PB-876), но проблема системная. На row нужно **два** элемента слева одновременно: чекбокс выбора **и** визуальный маркер (логотип бренда, иконка категории, аватар).

**Текущее состояние компонента:** `List Item / Left Side` поддерживает ровно один `Type` на ряд (Checkbox **или** Avatar **или** Icon). Нет варианта «checkbox + что-то ещё».

---

## Предложение

Расширить существующий `List Item / Left Side`, добавив composite-варианты:

| Новый variant | Состав | Кейсы |
|---|---|---|
| `Type=Checkbox + Avatar` | `[Checkbox 24]` + gap + `[Avatar 40]` | Выбор марок (с лого), выбор моделей с фото, выбор пользователей |
| `Type=Checkbox + Icon` *(на будущее)* | `[Checkbox 24]` + gap + `[Icon 24]` | Выбор категорий, опций с иконкой |

**Внутреннее устройство Left Side (Checkbox + Avatar):**

```
Left Side (HORIZONTAL auto-layout)
├── Checkbox 24×24
├── gap = row/gap-default (8)
└── Avatar 40×40
```

Высота — по высоте самого крупного элемента (40 для Avatar-варианта).

**Композиция всей row не меняется:** Left Side остаётся одним слотом с одной property `Type`. Для разработки — это всё ещё один `LeftSide`-композит, просто внутри две view'хи вместо одной.

---

## Brand logo — переиспользование Avatar

Brand logo (BMW, Mazda, Nissan и т.п.) **не вводится как отдельный атом**. Используется существующий `Avatar 40` с image-override на бренд-лого.

**Почему не отдельный атом `Brand Mark`:**
- Визуально Brand = Avatar (контейнер + image внутри). Структура идентична.
- Avatar 40 уже есть в DS с border и radius.
- Семантическая разница (человек vs бренд) — на уровне image-ассета, не компонента.
- Когда у Avatar появятся варианты Initials/Photo/Illustration — Brand станет ещё одним «контентом» того же контейнера, не отдельной сущностью.

**Параметры контейнера для бренд-лого:**

| Параметр | Значение | Зачем |
|---|---|---|
| Размер контейнера | 40×40 | Соответствует Avatar md, ритм с checkbox 24 + gap 8 |
| Inner image max | 32×32 | Safe area 4px по краям — лого не упирается в границы |
| Radius | `radius/full` (pill / circle) | Современные бренд-марки (BMW, Mazda, Mercedes) — круглые |
| Border | 1px `Border/Default` | Контейнер виден на белой карточке даже когда лого с прозрачным/белым фоном |
| Background | прозрачный | Не диктуем фон — это ответственность ассета |

**Альтернатива:** `radius/2` (8) rounded square. Универсальнее для wordmark-логотипов (Audi — 4 кольца, обрежется в круге). Решение принимаем по факту: если wordmark-проблемы всплывут — меняем на 8. Стартуем с pill.

**Если когда-нибудь Brand начнёт жить по своим правилам** (другие размеры, multi-logo grids, особая обработка wordmark'ов) — тогда выделим атом `Brand Mark`. Пока — Avatar.

## Почему не отдельный List Item

- Форк = двойная поддержка, расхождения токенов/стейтов со временем
- Сегодня — Brand. Завтра — `Checkbox + Icon`, `Radio + Avatar`, `Checkbox + Initials`. Через год — 8 параллельных листайтемов
- Правки самого List Item (новые стейты, изменения паддингов, новые токены) придётся применять везде вручную

## Что НЕ входит в компонент

- **Divider между rows** — НЕ часть List Item. Composition-уровень: дивайдер ставится снаружи между rows экрана, если нужен. По умолчанию — пустота 0 между rows, без дивайдера (см. `composition-rules.md` §7).

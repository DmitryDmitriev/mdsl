# Patterns

Композиционные паттерны Larixon Mobile DS — часто повторяющиеся куски интерфейсов, собранные из существующих компонентов.

## Что такое паттерн (и чем отличается от компонента)

| | Компонент | Паттерн |
|---|---|---|
| Где живёт | Figma library (UI-Kit-Mobile) + спека | Только спека (markdown) |
| Источник правды | Figma master + spec | Spec (markdown) |
| Сборка | `createInstance` от мастера | Композиция из существующих компонентов |
| Вариации | Жёстко через variant'ы | Гибко через выбор компонентов и параметров |
| Когда выбираем | Нужна параметризация через property panel, переиспользование с lock-in | Часто повторяющаяся композиция, но с гибкостью |

**Правило большого пальца:** если паттерн стабилизировался и дизайнеры пересобирают его 5+ раз вручную — рассмотреть превращение в компонент. До этого — паттерн.

## Список паттернов

| Category Grid | Меню подкатегорий внутри вертикали (Real estate, Motors, Electronics, …) | [category-grid.md](./category-grid.md) |

## Как добавить новый паттерн

1. Файл `<pattern-name>.md` в этой папке
2. Структура: «Когда применять» → «Анатомия» → «Токены» → «Состояния и edge-cases» → «Не делать»
3. Обновить таблицу в этом README
4. Если паттерн ссылается на правило композиции — линк в `composition-rules.md`

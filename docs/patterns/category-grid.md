# Category Grid

Меню подкатегорий внутри вертикали. 2-колоночная сетка кликабельных rows с label + counter. Используется на главных страницах вертикалей (Real estate, Motors, Electronics и т.п.).

## Когда применять

- Вертикаль имеет **≥ 4 подкатегорий**, которые удобно показать одновременно для быстрого выбора
- Каждая подкатегория имеет **counter** (количество объявлений)
- Цель — быстрая навигация на second level menu

**Если подкатегорий 3 или меньше** — стандартный 1-колоночный список из List Item, не нужна сетка.

**Если подкатегорий 12+** — показывать 8 самых популярных, остальное под кнопкой «Показать все» (см. §Trim & «Показать все»).

## Анатомия

```
Section card (composition-rules.md §2)
├── Heading «Real estate for sale»     ← composition-rules.md §3 (внутри карты)
│   ↕ section/title-to-content (8)
└── Grid (HORIZONTAL auto-layout, 2 колонки)
    ├── Column 1 (VERTICAL, FILL ширина):
    │   ├── Row: [Label] [Counter]
    │   ├── Row: [Label] [Counter]
    │   └── ...
    └── Column 2 (VERTICAL, FILL ширина):
        ├── Row: [Label] [Counter]
        └── ...
↕ section/gap (опционально, если ниже есть «Показать все»)
└── «Показать все» (Button Ghost / Link, FILL ширина)
```

**Каждая row** — List Item с конфигурацией:
- `Type = 1 str`
- Left Side: hidden (нет иконки/чекбокса)
- Content: label
- Right Side: trailing text only (counter), без chevron

См. [list-item-spec.md](../list-item-spec.md).

## Токены

| Параметр | Значение | Токен |
|---|---|---|
| Card padding | 16 | `section/padding-compact` |
| Heading → grid gap | 8 | `section/title-to-content` |
| Между колонками | 16 | `row/gap-default` × 2 — `screen/padding-horizontal` |
| Между rows в колонке | 0 | `list/item-gap` |
| Grid → «Показать все» (если есть) | 8 | `section/title-to-content` |
| Card radius | 12 | `radius/surface/surface` |

**Между колонками:** ширина gap'а = `screen/padding-horizontal` (16) — чтобы между двумя rows было то же визуальное «дыхание», что и от края экрана до начала контента. На 360-wide экране при card 360 без horizontal padding: 360 − 16(left pad) − 16(gap) − 16(right pad) = 312 / 2 = 156 на колонку.

## Распределение items по колонкам

**Заполнять колонки последовательно сверху вниз, слева направо:**

```
4 items: 2+2     6 items: 3+3     8 items: 4+4
A  B             A  D             A  E
C  D             B  E             B  F
                 C  F             C  G
                                  D  H
```

При нечётном количестве items — лишний в левой колонке:

```
5 items: 3+2     7 items: 4+3
A  D             A  E
B  E             B  F
C                C  G
                 D
```

## Trim & «Показать все»

Если подкатегорий > 8 (типичный лимит):

- Показываем **8 самых популярных** (отбор делает контент/CMS, не дизайн)
- Под сеткой — **Button Ghost** или **Link** «Показать все» (FILL ширина)
- Тап ведёт на полный плоский список (отдельный экран или раскрытие в этой же карте)

Если ≤ 8 — кнопки нет, показываем все.

Если ≤ 3 — не используем Grid вообще, обычный список (см. § Когда применять).

## Состояния и edge-cases

- **Длинный label** (например, «Загородная коммерческая недвижимость»): wrap на 2 строки, row растёт по высоте. Counter всегда right-aligned, не уезжает.
- **Очень длинный counter** (>1M, локализация типа «12 345 678»): на узких экранах counter может занять >50% ширины ячейки → label обрезается. Допустимо: counter — приоритетная информация.
- **Counter = 0**: показываем «0» (не скрываем категорию — категория есть, объявлений сейчас нет; пользователь может зайти и подписаться на появление).
- **Counter отсутствует** (никогда не считаем): row без trailing text, label занимает всю ширину.

## Не делать

- ❌ **Chevron справа в каждой row.** Tappability понятна из контекста (это меню), chevron'ы превратят 8 ячеек в визуальный шум.
- ❌ **3+ колонки на 360-wide.** Лейблы становятся нечитаемыми. Если хочется компактнее — рассмотреть chips-набор как альтернативу.
- ❌ **Иконки/лого в каждой row.** Это уровень навигации, не выбора. Иконки делают сетку тяжёлой и медленнее сканируется.
- ❌ **Сортировать по алфавиту, если есть «популярность».** Популярные категории сверху-слева — главный жест scan'а. Алфавит уместен только в полном списке за «Показать все».

## Прецеденты (что мы делаем как индустрия)

- **Avito (RU classifieds):** аналогичная 2-колоночная сетка на главной вертикали, counter справа, «Все категории» снизу
- **mobile.de:** 1 колонка с большими тач-таргетами + иконки (наша версия плотнее, без иконок)
- **Amazon Shop by Department:** 2 колонки с иконками + label, без counter (мы используем counter — это полезная информация для classifieds)

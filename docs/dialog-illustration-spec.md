# Диалог с иллюстрацией (DialogIllustration)

**Кодовое имя:** `DialogIllustration`.

**Первый тип** модального диалога для мобильного приложения: сверху **медиа** (иллюстрация или иконка), текст **по центру**, снизу **стек кнопок**.  
**Второй тип** (без картинки, левая выключка текста) — отдельная спека; рабочее имя для него: **`DialogPlain`** (текстовый диалог), чтобы не путать с `DialogIllustration`.

Используются только семантические токены: **docs/DESIGN-TOKENS.md**, **docs/COLOR-PALETTE.md**, **docs/TYPOGRAPHY.md**, кнопки — **docs/button-spec.md**.

---

## 1. Назначение

- Короткое сообщение с **визуальным акцентом** (онбординг, разрешения, успех/внимание, промо-одно действие).
- Одна смысловая группа: иллюстрация + заголовок + пояснение + 1–2 действия.

Не использовать для длинного текста без иллюстрации — там **`DialogPlain`**.

---

## 2. Структура

| Блок | Содержимое |
|------|------------|
| **Контент** | Слот медиа (центр) + колонка текста (центр): заголовок, описание |
| **Действия** | Вертикальный стек кнопок (как минимум primary; опционально secondary) |

Разделитель между контентом и кнопками в референсном макете **не обязателен** — достаточно вертикальных отступов блока кнопок. При необходимости единообразия с другими оверлеями допускается **divider/default** (1 px), **semantic.border.default**.

---

## 3. Панель диалога (карточка)

| Параметр | Токен / значение |
|----------|------------------|
| Фон | semantic.surface.primary |
| Скругление | **radius.overlay** (16 px) |
| Обводка | По умолчанию без; при необходимости **border/default**, semantic.border.default |
| Ширина на экране | С отступами от краёв экрана по правилам продукта (inset); внутри — на всю доступную ширину контейнера |

Подложка (backdrop) и поведение закрытия — общие для всех типов диалога: **semantic.background.overlay**, клик по подложке / Escape — по настройке экрана.

---

## 4. Блок контента

### 4.1 Внутренние отступы (padding)

От края карточки до медиа и текста — **внутренний inset компонента** → шкала **spacing** (не путать со **stack** как с отступами между секциями экрана).

| Сторона | Токен | px |
|---------|--------|-----|
| Все стороны блока «Контент» | **spacing/4** | 16 |

### 4.2 Медиа (иллюстрация, вариант «квадрат»)

Изображение в квадратном слоте **по центру**, ширина контейнера медиа — 100% ширины контентной области.

| Параметр | Значение |
|----------|-----------|
| Размер области отображения (квадрат) | **120×120 px** |
| Выравнивание | По центру по горизонтали |
| Обрезка | `object-fit: cover` или `contain` по продукту; сохранять визуальный центр |

**Примечание:** 120 px вне шага `spacing/N` — в коде допустима именованная константа слота иллюстрации (например `dialogIllustrationSize`).

Второй вариант медиа (на всю ширину контента, фиксированная высота) описывается отдельно при появлении в продукте.

### 4.3 Зазор «медиа → текст»

Между нижним краем блока медиа и блоком текста — вертикальный зазор между внутренними блоками:

| Параметр | Токен | px |
|----------|--------|-----|
| Gap | **stack-sm** | 16 |

### 4.4 Текст

Выравнивание: **по центру** (`text-align: center`). Ширина колонки — 100% контентной области.

| Роль | Стиль (TYPOGRAPHY.md) | px | Line height | Начертание | Цвет |
|------|------------------------|-----|-------------|------------|------|
| Заголовок | **H3** | 20 | 28 | Bold (700) | semantic.text.primary |
| Описание | **Body 1** | 16 | 24 | Regular (400) | semantic.text.secondary |

Зазор между заголовком и описанием:

| Параметр | Токен | px |
|----------|--------|-----|
| Gap | **spacing/2** | 8 |

---

## 5. Блок кнопок

| Параметр | Токен | px |
|----------|--------|-----|
| Padding горизонтальный | **spacing/4** | 16 |
| Padding вертикальный | **spacing/5** | 20 |
| Gap между кнопками | **spacing/3** | 12 |

Кнопки: **Primary** и **Secondary**, высота **48 px** → **height/md** (**docs/button-spec.md**, размер **md**). Ширина — на всю доступную ширину внутри горизонтального padding (полная ширина в блоке действий).

При необходимости более лёгкого вторичного действия допускается вариант **Ghost** / текстовая кнопка по отдельной договорённости — тогда уточнить отступы в макете.

---

## 6. Состояния

- **Закрыт** — не показывать или убрать из иерархии; фокус вернуть на триггер.
- **Открыт** — видимы подложка и карточка; фокус внутри диалога (trap focus, см. п. 7).

Анимация появления/исчезновения — по гайду продукта (например opacity + scale, 200–300 ms).

---

## 7. Доступность

- Контейнер модального окна: **role="dialog"**, **aria-modal="true"**.
- Связать заголовок: **aria-labelledby** (и при необходимости **aria-describedby** для описания).
- Закрытие по **Escape** (если включено для сценария).
- Управление фокусом: при открытии — на первую кнопку или логичный элемент; при закрытии — возврат на элемент, открывший диалог.

---

## 8. API (рекомендуемое)

| Prop / слот | Описание |
|-------------|----------|
| **open** | Открыт / закрыт |
| **onClose** | Запрос закрытия (backdrop, Escape, системная кнопка — по продукту) |
| **illustration** | Слот изображения / иконки (квадрат 120×120 по умолчанию) |
| **title** | Строка заголовка |
| **description** | Строка или узкий rich text описания |
| **primaryAction** | Подпись + обработчик (кнопка Primary, md) |
| **secondaryAction** | Опционально: подпись + обработчик (Secondary, md) |
| **children** | Альтернатива: кастомный футер вместо пары действий |

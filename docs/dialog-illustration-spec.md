# DialogIllustration — устарело

> **Статус: superseded.** Отдельного компонента `DialogIllustration` (и парного `DialogPlain`) больше нет.

С 2026-04-23 у нас **единый** компонент `Dialog` (Figma: ComponentSet `6632:39` в UI Kit Mobile). Кейс «диалог с иллюстрацией» собирается через свойства основного компонента — без отдельной спеки.

## Как делать «диалог с иллюстрацией» сейчас

| Старое решение | Новое решение |
|----------------|---------------|
| Отдельный компонент `DialogIllustration` | `Dialog` с `Image=true` |
| Иллюстрация 120×120 в padded-области | Иконка 140×140 в Image-зоне 328×180 (edge-to-edge) |
| Текст всегда по центру | Variant `Alignment=Center` (или `Left`) |
| Парный `DialogPlain` для текстовых | Тот же `Dialog` с `Image=false` |

Подмена иконки — через **Instance Swap** на placeholder `Bell_02` внутри Image-зоны (двойной клик → правая панель → swap).

## Размеры, токены, кнопки, доступность

См. **docs/dialog-spec.md** — все параметры (ширина 328, radius 16, padding контента, Buttons Stack, фокус-менеджмент) описаны там.

## История

- **2026-04-23.** Аудит модалок: 12 диалогов в 4 кластерах с зоопарком размеров. Принято решение об одном компоненте.
- **2026-04-25.** Image-зона увеличена со 120 до 180 px, иконка по умолчанию 140×140 прижата к низу. Добавлен `Alignment=Left/Center`. Старые компоненты (`Basic dialog`, `Dialog_IMG`, `img_container`) помечены `[deprecated]`.

# Спецификация компонента Dialog

Dialog — центрированное модальное окно поверх контента. Закрывается по подложке, по кнопкам действий или по Escape. Единый минимальный компонент с гибким контент-слотом.

Связанный паттерн (выезжающие панели): **docs/sheets-spec.md**.

---

## 1. Принципы

- **Один компонент.** Все вариации (Basic, с картинкой, с иконкой, с длинным скроллящимся контентом, notification) собираются из одного Dialog-компонента через наполнение контент-слота.
- **Слот-подход.** Заголовок, описание, картинка, иконка — всё внутри одного `Content` слота. Дизайнер сам решает композицию под кейс.
- **Кнопки — отдельный компонент.** Блок кнопок — инстанс `Buttons Stack` (см. button-spec.md), выбирается через swap-variant.
- **Без скролла как свойство.** Если контент высокий — дизайнер добавляет скролл сам внутри слота. Отдельного варианта нет.

---

## 2. Структура

```
Dialog
├── Image (optional, boolean property `withImage`) — edge-to-edge картинка/иллюстрация сверху
├── Content (slot) — заголовок, описание, иконка — любое наполнение в padded-области
└── Buttons Stack (instance, swap variant) — One / Two Horisontal / Two Vertical / Three Vertical и др.
```

### Свойства (component properties)

| Property | Type | Default | Описание |
|----------|------|---------|----------|
| **Image** | boolean | `false` | Показать edge-to-edge картинку сверху |
| **Content** | slot | — | Произвольный контент (заголовок + описание + иконка) |
| **Close** | boolean | `true` | Показать кнопку закрытия (если используется в контент-блоке с close) |

## 3. Токены

### 3.1 Подложка (backdrop)

| Параметр | Токен / значение |
|----------|------------------|
| Фон | `semantic.background.overlay` |
| Поведение | Клик по подложке закрывает диалог (опционально, по умолчанию `true`) |

### 3.2 Панель (panel)

| Параметр | Токен / значение |
|----------|------------------|
| Фон | `semantic.surface.primary` |
| **Ширина** | **328 px** (= 360 − 16×2, отступы по 16 px с каждой стороны на минимальном экране 360) |
| **Радиус** | **16 px** на всех 4 углах (`radius.overlay`) |
| Тень | не применяется (диалог идёт поверх overlay — тень не нужна) |
| Позиционирование | По центру экрана (X = (vw − 328) / 2), вертикально — по центру или по верхней трети экрана |

### 3.3 Image (при withImage=true)

| Параметр | Токен / значение |
|----------|------------------|
| Ширина | 328 px (edge-to-edge, без горизонтальных отступов) |
| Высота | 120 px (рекомендуемая; допустима кастомизация под сценарий) |
| Скругление | Наследуется от корня Dialog (верхние углы 16, нижние 0) через `clipsContent` |
| Позиция | Первый ребёнок, над Content |

### 3.4 Content (slot)

| Параметр | Токен / значение |
|----------|------------------|
| Padding | stack-md (24 px) по вертикали, spacing/4 (16 px) по горизонтали |
| Gap между блоками | stack-sm (16 px) — по смыслу |
| Цвет заголовка | `semantic.text.primary`, typography: 16–18 px weight 600 |
| Цвет описания | `semantic.text.secondary`, typography: 14 px weight 400 |
| Иконка (без `withImage`) | Маленькая, 24–40 px, выравнивание по центру по горизонтали, отступ снизу stack-sm (16 px) |

### 3.5 Buttons Stack (instance)

| Параметр | Значение |
|----------|----------|
| Компонент | `Buttons Stack` (см. button-spec.md) |
| Варианты | `One`, `Two Horisontal`, `Two Horisontal Reverce`, `Two Vertical`, `Three Vertical` |
| Поведение | Подменяется через instance swap в Figma; в коде — соответствующий prop |

## 4. Состояния

- **Закрыт** — диалог скрыт (не рендерится или opacity: 0, pointer-events: none).
- **Открыт** — диалог видим, подложка затемнена, фокус внутри диалога.

Анимация: fade-in + scale 0.95 → 1 для панели, fade-in для подложки. Длительность 150–200 ms, easing ease-out.

---

## 5. Доступность

- `role="dialog"` и `aria-modal="true"` на панели
- `aria-labelledby` на заголовок (если заголовок есть в Content)
- `aria-describedby` на описание (если описание есть)
- Фокус при открытии — на первый фокусируемый элемент или на primary-кнопку
- Trap focus внутри диалога; при закрытии — возврат фокуса на триггер
- Закрытие по Escape

---

## 6. API компонента (рекомендуемое)

- **open** (boolean) — открыт / закрыт
- **onClose** (function) — вызывается при запросе закрытия (клик по backdrop, Escape, кнопка действия)
- **closeOnBackdropClick** (boolean, default `true`)
- **children** — контент диалога (в Content-слот)
- **buttonsStack** — компонент блока кнопок (обычно prop `actions` или через children слот)

---

## 7. Правила применения

1. **Диалог vs Sheet.** Диалог — для важных действий, требующих подтверждения (удаление, выход, ошибка). Sheet — для выбора, фильтров, длинных форм. Если контент высокий — используй Sheet.
2. **Не разносить информацию на уровни.** Если один диалог превращается в цепочку — пересмотри логику: возможно, это Sheet или отдельный экран.
3. **Всегда показывай хотя бы одну кнопку.** Диалог без действия — это уведомление, а не диалог.
4. **Текст заголовка** — короткий, глагольный («Удалить объявление?»).
5. **Текст описания** — объясняет последствия действия.
6. **Primary-кнопка** — справа в горизонтальном стеке (или снизу в вертикальном); выделяется цветом `Accent/Primary`.

---

## 8. Deprecated

После объединения в единый компонент удалены из активного использования:

- **`Basic dialog`** (ComponentSet, 312×H, r=28) — `[deprecated]`. Ширина 312 и радиус 28 не соответствовали системе. Мигрируй на `Dialog`.
- **`Dialog_IMG`** (Component, 328×340, r=28) — `[deprecated]`. Картинка теперь добавляется в Content-слот вручную. Мигрируй на `Dialog` с изображением внутри слота.

В новых макетах использовать только канонический `Dialog` (id 4701:462 в UI Kit Mobile).

## 10. История решений

**2026-04-23 — аудит модалок.** Скан продуктового файла показал 12 диалогов в 4 визуальных кластерах с зоопарком размеров (312/328/344) и радиусов (16/24/28/32). Приняты решения:

1. **Единый компонент** — никаких вариантов вроде «Basic / Editable / IMG». Всё через Content-слот.
2. **Ширина 328** (не 312) — чтобы отступы от краёв экрана 360 составляли по 16 px, совпадая с системным padding
3. **Радиус 16** (не 28, как было в `Basic dialog`) — унификация с sheet-радиусом `radius.overlay`
4. **Scroll не вариант** — добавляется вручную в Content, если нужен
5. **Кнопки — инстанс `Buttons Stack`** (не пересобираем в диалоге)

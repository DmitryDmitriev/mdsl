# Stories — спецификация компонента

Полноэкранный story-вьюер (Instagram/Snapchat-паттерн): медиа на весь экран + накладываемый chrome (прогресс-бар сегментов, close, заголовок, реплай/CTA). Кросс-платформенный.

**Figma:** страница **🟢 Stories** (`4535:6698`). Building blocks: `Img Conteiner` (`4535:6775`), `Top Progress bar` (`4535:6736`, варианты Type=Empty/1/2/3/4), `Progress Indicator` (`4535:6770`, Active=Yes/No), `Stroies Constructor` (`4535:6732`).

**Категория:** Special / System · full-screen overlay.

> **Зачем эта спека.** Заготовка в Figma собрана под один фиксированный размер `360×800` и не описывает (1) поведение под **разные пропорции экрана** и (2) **safe-area** — зоны, в которых можно безопасно класть тексты и значимые элементы поверх медиа. Эти два блока — §4 и §5 — основное содержание документа.

---

## 1. Обзор

Story = **полноэкранное иммерсивное медиа** (фото/видео), поверх которого лежит прозрачный слой управления:

- **сверху** — сегментный прогресс (сколько story в ленте + позиция в текущем), close, заголовок (аватар + автор + время);
- **снизу** — реплай-поле / CTA / действия (продуктовый слот);
- **центр** — медиа + накладываемый продуктовый контент (подписи, стикеры, ссылки).

Ключевой принцип: **медиа тянется на весь экран устройства** (edge-to-edge, под вырез и под жестовую полоску), а **chrome привязан к safe-area экрана, а не к медиа**. Размер `360×800` в Figma — baseline для дизайна, **не** фиксированный размер компонента.

## 2. Варианты (variants)

### Top Progress bar — `Type`

| Type | Сегментов | Когда |
|---|---|---|
| `Empty` | 0 (только close) | Одиночная story без сегментации / заглушка |
| `1` | 1 | Один story-кадр |
| `2` / `3` / `4` | 2 / 3 / 4 | Иллюстративные позиции для дизайн-ревью |
| `N` (код) | динамически | В коде число сегментов = числу кадров в ленте автора |

> Type=1..4 в Figma — **иллюстративные** (как у [Progress](./progress-spec.md): варианты для ревью, не функциональные значения). В коде число сегментов и активный индекс приходят из данных.

### Progress Indicator (один сегмент) — `Active`

| Active | Состояние | Заливка — канон (story) | В заготовке (поправить) |
|---|---|---|---|
| `No` | непросмотренный / остаток | **`White 40`** (белый @ 40%) | `Border/Default` ⚠️ |
| `Yes` | активный (текущий) | **`White/Main`** (#FFFFFF) | `Border/Active` ⚠️ |

> **Палитра бара — белая, theme-invariant.** Сегменты лежат поверх произвольной медиа, поэтому красятся фиксированными светлыми токенами из White-шкалы (`White/Main` / `White 40`), а **не** theme-adaptive `Border/*` — иначе в Dark активный сегмент инвертируется в тёмный и тонет. Заготовка пока на списочных `Border/Active`/`Border/Default` — поправить в Figma. Подложка-страховка — top scrim (§5.3).

Полная модель состояний сегмента (для кода, в Figma пока не разнесено по variant'ам):

| Состояние | Значение |
|---|---|
| **Seen** (просмотренный, до активного) | заполнен на 100% |
| **Active** (текущий) | заполняется анимированно 0→100% за длительность кадра |
| **Unseen** (после активного) | пустой track |

## 3. Структура слоёв

```
Story (FULL-BLEED, 100% × 100% экрана устройства)
├── Media               ← фото/видео, object-fit: cover, центрировано (§4)
├── Scrim/top           ← градиент сверху для легибельности chrome (§5.3)
├── Scrim/bottom        ← градиент снизу для легибельности реплай/CTA (§5.3)
├── Top chrome          ← привязан к safe-area-inset-top (§5)
│   └── Top Progress bar (flex-col, items-end, w=FILL)
│       ├── Progress (HORIZONTAL, gap=spacing/1 (4), padding-x=spacing/4 (16))
│       │   └── Progress Indicator[]  ← flex-1 (равная ширина), bar h=4, radius=pill
│       └── Close (justify-end, padding-x=spacing/1 (4))
│           └── state-layer (p=spacing/3 (12)) → Icon 24×24  ⇒ touch 48×48
│   └── Story header (slot)   ← аватар 32 + автор + время + more; gap до бара spacing/2 (8)
├── Content-safe band   ← продуктовый контент: подписи, стикеры, значимые элементы (§5.2)
└── Bottom chrome (slot)← реплай-поле / CTA / действия, привязан к safe-area-inset-bottom (§5)
```

`Top Progress bar` стекается вертикально: ряд сегментов (12 px) → ряд close (48 px). Высота: Type=Empty = 48, Type=1..4 = 60.

## 4. Адаптивность под пропорции экрана

Заготовка прибита к `360×800` (≈ 9:20). Реальные экраны различаются: от 16:9 (0.562, бюджетные/старые) до ~21:9 (0.43, высокие Android), iPhone 14/15 ≈ 19.5:9 (0.46). Компонент **не fixed-size** — он заполняет экран.

### 4.1 Медиа-канвас

| Правило | Значение |
|---|---|
| Размер канваса | **100% ширины × 100% высоты экрана устройства** (edge-to-edge, включая зону под вырезом и под жестовой полоской) |
| Отрисовка медиа | **`cover`** — масштабируется до полного заполнения, центрируется, излишек обрезается равномерно. Без letterbox, без полей |
| Baseline дизайна | `360×800` (для макетов; не ограничение рантайма) |
| Авторский content-safe | База авторинга **360×800** (9:20), гарантированная text-safe зона **320×424 dp** по центру — см. §5.4. Значимый контент держать в ней, чтобы пережил кроп на всех пропорциях (вплоть до 16:9) |

**Почему cover, а не contain.** Story — иммерсивный формат; поля/леттербокс ломают ощущение полного экрана. Кроп компенсируется тем, что весь значимый контент лежит в content-safe band (§5.2), которая заведомо видна.

### 4.2 Поведение chrome под разные пропорции

Chrome **не привязан к медиа** — он привязан к экрану и его safe-area:

- **Top Progress bar + close + header** — прижаты к верху, отступ от `safe-area-inset-top` (§5). На высоком экране бар не «уезжает» от выреза, на низком — не наезжает на статус-бар.
- **Bottom chrome (реплай/CTA)** — прижат к низу, отступ от `safe-area-inset-bottom`. На устройствах с жестовой полоской поднимается над ней, на кнопочной навигации — садится ниже.
- **Боковые** — значимый контент инсетится на `screen/padding-horizontal` (16) от краёв + учёт скруглений/кривизны экрана.

Итог: при любой пропорции прогресс-бар читается сразу под вырезом, реплай — над жестовой полоской, а центральная медиа-зона тянется на весь остаток.

## 5. Safe-area — зоны для текстов и значимых элементов

Две независимые «безопасности»: **пространственная** (не столкнуться с системным UI и chrome) и **легибельная** (читаемость поверх произвольной медиа).

### 5.1 Пространственные зоны (сверху вниз)

| Зона | Граница | Что внутри / правило |
|---|---|---|
| **System top inset** | `env(safe-area-inset-top)` · fallback **44** | Статус-бар / вырез / Dynamic Island. Контент не класть |
| **Top chrome** | inset-top → +60 | Progress bar (12) + close (48). Отступ прогресса от inset-top = `spacing/2` (8). Тут только сам бар |
| **Header (опц.)** | под top chrome, gap `spacing/2` (8) | Аватар 32 + автор + время + more. Продуктовый слот |
| **Content-safe band** | top: (inset-top + top chrome + header) + `spacing/2`<br>bottom: (inset-bottom + bottom chrome) + `spacing/2`<br>left/right: **16** (`screen/padding-horizontal`) | **Зона гарантированной видимости** для накладываемых текстов, подписей, стикеров, значимых элементов. Всё значимое — сюда |
| **Bottom chrome (опц.)** | над system bottom inset | Реплай-поле / CTA / действия. Продуктовый слот |
| **System bottom inset** | `env(safe-area-inset-bottom)` · fallback iOS **34** / Android 0–48 | Жестовая полоска (Home Indicator) / системная навигация. Тапабельный контент не класть |

> **Content-safe band — основной ответ дизайна.** Любой текст или значимый элемент, чья обрезка/перекрытие недопустимы, должен лежать в этой зоне. Она автоматически отступает от выреза, прогресс-бара, реплая и жестовой полоски на всех пропорциях.

### 5.2 Минимальные тач-зоны

- Close — **48×48** (`size/xl`: icon 24 + state-layer `spacing/3`=12). Уже в заготовке.
- Любой интерактив в content-safe / bottom chrome — ≥ **48×48** (WCAG/HIG).
- Тапы навигации по кадрам (лево/право/долгий тап-пауза) — на медиа-слое, под chrome; chrome перехватывает тапы первым.

### 5.3 Легибельность поверх произвольной медиа

Медиа непредсказуема (светлая/тёмная/пёстрая) — chrome и тексты нужно защитить:

| Приём | Спецификация |
|---|---|
| **Top scrim** | Вертикальный градиент сверху: `Background/Overlay` 0% → ~40% на высоту `inset-top + top chrome + ~24`. Гарантирует читаемость прогресс-бара, close, header |
| **Bottom scrim** | Зеркальный градиент снизу под bottom chrome (если есть реплай/CTA) |
| **Прогресс-бар — белая палитра** | Активный = **`White/Main`** (#FFFFFF), трек = **`White 40`** (белый @ 40%). Оба из White-шкалы — **theme-invariant**, не инвертируются в Dark. Заготовку с `Border/Active`/`Border/Default` поправить. + top scrim как страховка над светлой медиа |
| **Тексты** | Поверх медиа — цвет `Text&Icon` «White applied» (`White/Main`, theme-invariant) на scrim или с собственной подложкой. Не полагаться на «обычно тёмное фото» |

### 5.4 Гайд для креативов (дизайн коммуникации, PNG)

Для дизайнера, который рисует **PNG-креатив**. Он не управляет рантайм-`env(safe-area)`, поэтому зоны зашиваются **фиксированными числами** (концретизация §5.1 под статичный PNG).

**База авторинга:** `360×800 dp` (9:20, мин. ширина Android) · экспорт **×3 = `1080×2400 px`**.

| Пропорция | dp | px @3x | Девайс |
|---|---|---|---|
| 16:9 | 360×640 | 1080×1920 | старые / бюджет |
| 18:9 | 360×720 | 1080×2160 | 2:1 |
| 19.5:9 | 360×780 | 1080×2340 | iPhone X–15 |
| **20:9** | **360×800** | **1080×2400** | **база · Android** |
| 21:9 | 360×840 | 1080×2520 | высокие Android |

**Фиксированные поля (одинаковы во всех пропорциях):**

| Поле | dp | px @3x | Что закрывает |
|---|---|---|---|
| Верхний chrome keep-out | 120 | 360 | статус-бар + прогресс + close + место под header |
| Нижний chrome keep-out | 96 | 288 | реплай / CTA + жесты |
| Боковое поле | 20 | 60 | `screen/padding` 16 + запас под боковой срез ~9 на 21:9 |
| **Гарантированная text-safe зона** | **320×424** | **960×1272** | по центру по ширине, отступ сверху 120. Видна даже на 16:9 |

**Правила:**

1. **Фон** — на весь кадр до краёв (терпит срез).
2. **Текст и лого** — только в text-safe зоне.
3. База 9:20 → на «низких» экранах (16:9) срезается **верх и низ** (~80 dp каждый), бока почти не режутся. Поэтому критичное — по центру, не у верх/низ кромки.
4. Светлая «extra» зона ниже text-safe — бонус-место на высоких экранах; на 16:9 срежется, критичное туда не ставить.

**Шаблон:** нативные фреймы под все 5 пропорций — Figma UI-Kit-Mobile, страница **🟢 Stories** (ниже заготовки `4535:6698`): по фрейму на пропорцию с одинаковой text-safe зоной, chrome-полосами и подписями dp + px.

## 7. Доступность (a11y)

- **Close** — 48×48, `aria-label="Закрыть"`.
- **Прогресс** — `role="progressbar"` на активном сегменте (`aria-valuenow` 0..100); семантика «кадр N из M».
- **Навигация** — лево/право-тап (предыдущий/следующий), долгий тап — пауза. Жесты дублируются доступными контролами для screen-reader (предыдущий/следующий/пауза).
- **Контраст** — chrome и тексты поверх медиа: гарантировать ≥ 4.5:1 за счёт scrim'ов (§5.3), не за счёт удачного фото.
- **Reduce motion** — анимацию заполнения сегмента и авто-переход уважают системный `prefers-reduced-motion`: переход по тапу, без бегущей анимации.

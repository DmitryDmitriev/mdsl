# Правила использования переменных (Design Tokens)

## Структура токенов

Переменные разделены на два уровня:

- **Core** — примитивные значения (числовая шкала). Не используются напрямую в экранах и компонентах.
- **Semantic** — семантические токены, ссылающиеся на Core. Используются во всех макетах и в коде.

В UI применяются только токены из **Semantic**.

---

## Core

### Spacing

- Базовая единица: **4 px**.
- Токен `spacing/N` = **4 × N** px.
- Индексы шкалы: 0, 1, 2, 3, 4, 5, 6, 8, 10, 12, 14, 16.

| Token       | Значение (px) |
|-------------|---------------|
| spacing/0   | 0             |
| spacing/1   | 4             |
| spacing/2   | 8             |
| spacing/3   | 12            |
| spacing/4   | 16            |
| spacing/5   | 20            |
| spacing/6   | 24            |
| spacing/8   | 32            |
| spacing/10  | 40            |
| spacing/12  | 48            |
| spacing/14  | 56            |
| spacing/16  | 64            |

### Radius

- Токен `radius/N` = **4 × N** px для N = 0…5.
- `radius/full` = 999 px (полное скругление, pill/круг).

| Token       | Значение (px) |
|-------------|---------------|
| radius/0    | 0             |
| radius/1    | 4             |
| radius/2    | 8             |
| radius/3    | 12            |
| radius/4    | 16            |
| radius/5    | 20            |
| radius/full | 999           |

### Border (толщина линии)

- Шкала в **пикселях**, шаг 1 px (для обводок и разделителей).
- Токены толщины бордера и дивайдеров.

| Token      | Значение (px) |
|------------|---------------|
| border/0   | 0             |
| border/1   | 1             |
| border/2   | 2             |
| border/3   | 3             |

## Semantic

### space

Внутренние отступы и зазоры внутри компонентов (padding, gap между иконкой и текстом, между чипами и т.п.).

- Использовать для: padding кнопок, полей, карточек; gap в auto-layout.
- Не путать с **stack** — stack для вертикальных отступов между блоками.

### stack

Вертикальные отступы между секциями и блоками контента (между карточками, между заголовком и списком и т.д.).

- **stack-sm** → spacing/4 (16 px)
- **stack-md** → spacing/6 (24 px)
- **stack-lg** → spacing/8 (32 px)
- **stack-xl** → spacing/12 (48 px)
- **stack-2xl** → spacing/16 (64 px)

### height

Минимальная высота интерактивных контролов. Один размер — для кнопок, однострочных полей ввода и селектов одного размера.

| Токен   | Референс   | Значение | Использование                    |
|---------|------------|----------|----------------------------------|
| height/sm | spacing/10 | 40 px  | Маленькие кнопки, компактные инпуты |
| height/md | spacing/12 | 48 px  | Стандартные кнопки и инпуты      |
| height/lg | spacing/14 | 56 px  | Крупные CTA, большие инпуты      |

- В компонентах: фиксированная высота фрейма привязана к переменной (height/sm, height/md или height/lg).
- Для текстарии используется отдельный минимум (см. ниже), не height/sm|md|lg.

### radius

Семантика скруглений:

- **control-sm**, **control-md**, **control-lg** — кнопки, инпуты, чипы (размер по контексту).
- **surface** — карточки, панели.
- **overlay** — модалки, bottom sheet.
- **pill** — pill-кнопки, чипы-капсулы.

### border и divider (толщина)

| Токен              | Референс  | Использование |
|--------------------|-----------|----------------|
| border/default     | border/1  | Обычные бордеры (карточки, инпуты, чипы) |
| border/emphasis    | border/2  | Фокус, выделение, акцент |
| divider/default    | border/1  | Разделители между блоками и строками списка |

- Цвет линии задаётся отдельными цветовыми токенами (color/border, color/divider).

## Правила использования

1. **В макетах и компонентах** используйте только переменные из **Semantic**. Core не подставлять в экраны вручную.
2. **Отступы**: внутри компонентов — **space**; между блоками по вертикали — **stack**.
3. **Высота контролов**: у кнопок и однострочных инпутов — **height/sm**, **height/md** или **height/lg**. Один размер в одном контексте (например, основная кнопка и поле ввода в одной форме — height/md).
4. **Скругления**: выбирать из **radius** по смыслу (control-*, surface, overlay, pill), не вводить произвольные значения.
5. **Текстария (многострочное поле)**: минимальная высота задаётся отдельным токеном (например height/textarea-min), не height/sm|md|lg. Фиксированная высота у текстарии не используется — только минимум и рост по контенту.
6. **Бордеры и дивайдеры**: толщина линии — только **border/default** (1 px) или **border/emphasis** (2 px). Дивайдеры между блоками — **divider/default** (1 px). Не использовать произвольные значения (например 1 px вручную).

---

## Синхронизация с кодом (Compose)

- В коде те же значения: spacing/N → `N * 4.dp`, radius/N → `N * 4.dp`, height/sm|md|lg → 40/48/56 dp, border/1 → 1.dp, border/2 → 2.dp.
- Семантические имена в коде должны соответствовать этой таблице (например `ControlHeight.Medium` = 48.dp).

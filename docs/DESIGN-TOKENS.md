# Правила использования переменных (Design Tokens)

## Структура токенов

Переменные разделены на два уровня:

- **Core** — примитивные значения (числовая шкала). Не используются напрямую в экранах и компонентах.
- **Semantic** — семантические токены, ссылающиеся на Core. Используются во всех макетах и в коде.

В UI применяются только токены из **Semantic**.

---

## Core

### Spacing

- Базовая единица: **4 px**.
- Токен `spacing/N` = **4 × N** px.
- Индексы шкалы: 0, 1, 2, 3, 4, 5, 6, 8, 10, 12, 14, 16.

| Token       | Значение (px) |
|-------------|---------------|
| spacing/0   | 0             |
| spacing/1   | 4             |
| spacing/2   | 8             |
| spacing/3   | 12            |
| spacing/4   | 16            |
| spacing/5   | 20            |
| spacing/6   | 24            |
| spacing/8   | 32            |
| spacing/10  | 40            |
| spacing/12  | 48            |
| spacing/14  | 56            |
| spacing/16  | 64            |

### Radius

- Токен `radius/N` = **4 × N** px для N = 0…5.
- `radius/full` = 999 px (полное скругление, pill/круг).

| Token       | Значение (px) |
|-------------|---------------|
| radius/0    | 0             |
| radius/1    | 4             |
| radius/2    | 8             |
| radius/3    | 12            |
| radius/4    | 16            |
| radius/5    | 20            |
| radius/full | 999           |

### Border (толщина линии)

- Шкала в **пикселях**, шаг 1 px (для обводок и разделителей).
- Токены толщины бордера и дивайдеров.

| Token      | Значение (px) |
|------------|---------------|
| border/0   | 0             |
| border/1   | 1             |
| border/2   | 2             |
| border/3   | 3             |

## Semantic

### space

Внутренние отступы и зазоры внутри компонентов (padding, gap между иконкой и текстом, между чипами и т.п.).

- Использовать для: padding кнопок, полей, карточек; gap в auto-layout.
- Не путать с **stack** — stack для вертикальных отступов между блоками.

### stack

Вертикальные отступы между секциями и блоками контента (между карточками, между заголовком и списком и т.д.).

- **stack-sm** → spacing/4 (16 px)
- **stack-md** → spacing/6 (24 px)
- **stack-lg** → spacing/8 (32 px)
- **stack-xl** → spacing/12 (48 px)
- **stack-2xl** → spacing/16 (64 px)

### height

Минимальная высота интерактивных контролов. Один размер — для кнопок, однострочных полей ввода и селектов одного размера.

| Токен   | Референс   | Значение | Использование                    |
|---------|------------|----------|----------------------------------|
| height/xs | spacing/8  | 32 px  | Компактные кнопки (xs)          |
| height/sm | spacing/10 | 40 px  | Маленькие кнопки, компактные инпуты |
| height/md | spacing/12 | 48 px  | Стандартные кнопки и инпуты      |
| height/lg | spacing/14 | 56 px  | Крупные CTA, большие инпуты      |

- В компонентах: фиксированная высота фрейма привязана к переменной (height/xs, height/sm, height/md или height/lg).
- Для текстарии используется отдельный минимум (см. ниже), не height/sm|md|lg.

### radius

Семантика скруглений (алиасы на Core как в Figma, коллекция Semantic):

| Semantic | Core (референс) | px | Использование |
|----------|-----------------|-----|----------------|
| control-sm | radius/1 | 4 | Мелкие контролы |
| control-md | radius/2 | 8 | Кнопка **xs**, rounded badge, мелкие кнопки-иконки |
| control-lg | radius/3 | 12 | Кнопки **sm**, **md**, **lg** |
| surface | radius/3 | 12 | Карточки, панели |
| overlay | radius/4 | 16 | Модалки, bottom sheet (открытый край) |
| pill | radius/full | 999 | Капсулы, чипы-pill |

### border и divider (толщина)

| Токен              | Референс  | Использование |
|--------------------|-----------|----------------|
| border/default     | border/1  | Обычные бордеры (карточки, инпуты, чипы) |
| border/emphasis    | border/2  | Фокус, выделение, акцент; **Outline-кнопка** sm/md/lg (см. docs/buttons-spec-figma.md) |
| divider/default    | border/1  | Разделители между блоками и строками списка |

- Цвет линии задаётся отдельными цветовыми токенами (color/border, color/divider).
- **Кнопка Outline:** высота **32 (xs)** → толщина **border/default** (1 px); **40 / 48 / 56** → **border/emphasis** (2 px). Реализация: `borderWidth` в `@/tokens` (spacing.ts).

### typography (подписи, caption)

Токены подписей и мелкого текста (подробнее — **docs/TYPOGRAPHY.md**):

| Токен | Размер (px) | Line height (px) | Начертание |
|-------|-------------|------------------|------------|
| typography/caption-lg | 14 | 16 | Regular, Medium, Bold |
| typography/caption-md | 12 | 16 | Regular, Medium, Bold |
| typography/caption-sm | 10 | 12 | Medium |

Использовать в таблицах токенов компонентов для строки «Текст» (например typography/caption-lg, color/text-primary).

### Иконки

- **Источник:** иконки для компонентов берутся только из библиотеки Figma [Assets](https://www.figma.com/design/iRb5cHw514oCpucz44gjGv/Assets?node-id=2-4).
- **Размеры:** в компонентах используются иконки только трёх размеров — **16**, **24** и **32** px. Других размеров не вводить.
- В таблице токенов компонента для иконок указывать: размер (16 / 24 / 32) и цвет (семантический токен).

## Правила использования

1. **В макетах и компонентах** используйте только переменные из **Semantic**. Core не подставлять в экраны вручную.
2. **Отступы**: внутри компонентов — **space**; между блоками по вертикали — **stack**.
3. **Высота контролов**: у кнопок и однострочных инпутов — **height/xs** (32), **height/sm** (40), **height/md** (48) или **height/lg** (56). Один размер в одном контексте (например, основная кнопка и поле ввода в одной форме — height/md).
4. **Скругления**: выбирать из **radius** по смыслу (control-*, surface, overlay, pill), не вводить произвольные значения.
5. **Текстария (многострочное поле)**: минимальная высота задаётся отдельным токеном (например height/textarea-min), не height/sm|md|lg. Фиксированная высота у текстарии не используется — только минимум и рост по контенту.
6. **Бордеры и дивайдеры**: толщина линии — только **border/default** (1 px) или **border/emphasis** (2 px). Дивайдеры между блоками — **divider/default** (1 px). Не использовать произвольные значения (например 1 px вручную).
7. **Иконки**: использовать только иконки из библиотеки Figma Assets (см. ссылку в разделе «Ссылки»); в компонентах — только размеры **16**, **24** и **32** px.

8. **Спецификация компонента для Figma**: для каждого компонента вести **чёткую таблицу токенов** — все значения берутся только из неё при сборке в Figma. В таблице указывать:
   - **Отступы** (padding, gap): токены space / stack по ролям (например padding-horizontal, padding-vertical, gap между иконкой и текстом).
   - **Радиусы скругления**: токен radius (control-sm, surface и т.д.).
   - **Толщина бордера**: токен border/default или border/emphasis (если есть обводка).
   - **Текст**: размер/стиль (типографический токен) и цвет (семантический цветовой токен).
   - **Иконки**: размер — только **16**, **24** или **32** px; источник — Figma Assets (см. ссылку в разделе «Иконки»); цвет — семантический токен.
   - **Высота** (для контролов): height/xs, height/sm, height/md, height/lg.

   Иконки в таблице — только 16 / 24 / 32 px из Figma Assets.

   **Пример строк таблицы:**

   | Параметр           | Токен / значение   |
   |--------------------|--------------------|
   | Padding horizontal | space-md           |
   | Padding vertical   | space-sm           |
   | Gap (icon–text)    | space-sm           |
   | Radius             | control-md         |
   | Border             | border/default     |
   | Высота             | height/md          |
   | Текст              | typography/caption-lg, color/text-primary |
   | Иконка             | 24 px (Figma Assets), color/on-surface |

   Без такой таблицы компонент не считается описанным; сборка в Figma — строго по токенам из таблицы.

9. **Один компонент — одна спека.** Для каждого компонента ведётся один файл спецификации (один .md на компонент). Не создавать отдельные «описание», «description» или вторую спеку для того же компонента; вся информация — в одной спеки.

10. **Порядок для новых компонентов:** сначала **спека** в `docs/` (таблица токенов по п. 8), синхронно с **Variables** в Figma; затем реализация в коде и Storybook. Глобальные правила — в этом файле и в **TYPOGRAPHY** / **COLOR-PALETTE**; детали контрола — в спеке компонента (пример: **docs/buttons-spec-figma.md**).

## Синхронизация с кодом (Compose)

- В коде те же значения: spacing/N → `N * 4.dp`, radius/N → `N * 4.dp`, height/xs|sm|md|lg → 32/40/48/56 dp, border/1 → 1.dp, border/2 → 2.dp.
- Семантические имена в коде должны соответствовать этой таблице (например `ControlHeight.Medium` = 48.dp).

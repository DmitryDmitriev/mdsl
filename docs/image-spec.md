# Image — спецификация компонента

Медиа-бокс под фотографию для листинг-карточек, гридов и галерей. Контейнер с фиксированным соотношением сторон, скруглением и обрезкой; фото подставляется как image fill. Использует семантические токены из **docs/DESIGN-TOKENS.md**.

**Figma:** файл **Larixon Assets** (`iRb5cHw514oCpucz44gjGv`), страница **Photo Library**, набор **Image** (COMPONENT_SET, `2732:26`).

**Категория:** Atom · Media. **Расположение:** живёт в файле Assets рядом с фотопулом (решение Дмитрия 2026-06-29 — «фото-бокс и фото в одном месте»), а не в основной библиотеке компонентов. Скругление привязано к импортированной переменной `radius/surface/surface` из подписанной библиотеки токенов.

---

## 1. Обзор

`Image` — простейший атом-носитель растрового медиа: прямоугольник с image fill (`scaleMode = FILL`, центр-кроп), скруглением `radius/surface` (12) и `clipsContent`. Один variant-проп — **Ratio** (соотношение сторон). Больше у компонента ничего нет: ни состояний, ни overlay-элементов, ни рамок (см. §5 «Чего здесь нет»).

Дефолтное фото каждого варианта берётся из internal-пула `_Claude_/image-pool/` (placeholder, Bazaraki UGC — только для internal-дизайна). В продуктовом макете дизайнер заменяет фото через **override fill** на инстансе.

---

## 2. Варианты (variants)

| Свойство | Значения | Default size (px) |
|----------|----------|-------------------|
| **Ratio** | `1:1`  | 240 × 240 |
| **Ratio** | `4:3`  | 240 × 180 |
| **Ratio** | `3:2`  | 240 × 160 |
| **Ratio** | `16:9` | 240 × 135 |
| **Ratio** | `3:4`  | 240 × 320 |

Итого **5 вариантов**. Базовая ширина 240 — это дефолт превью; инстанс **свободно ресайзится** под нужный размер контейнера, fill сохраняет кроп. Соотношение восстанавливается вручную (variants задают пропорцию, не жёсткий размер).

Покрытие пула: 1:1 / 4:3 / 3:2 / 16:9 — landscape-семья; 3:4 — портрет (телефоны/часть авто).

---

## 3. Структура слоёв

```
Image (COMPONENT_SET) ← HORIZONTAL auto-layout (раскладка вариантов)
  └── Ratio=<value> (COMPONENT)   ← image fill, clipsContent
```

- **Вариант (COMPONENT):** одиночный фрейм, `fills = [{ type: IMAGE, scaleMode: FILL, imageHash }]`, `clipsContent = true`.
- Никаких дочерних нод — фото живёт в `fills` корня. Замена = override fill на инстансе.

## 4. Таблица токенов

| Параметр | Значение | Токен | Примечание |
|----------|----------|-------|------------|
| Скругление (4 угла) | 12 | `radius/surface/surface` (`VariableID:4334:72`) | Совпадает с radius карточек/Skeleton Listing Card |

Цветовых токенов нет: компонент полностью закрыт медиа. Если фото отсутствует — это **не** этот компонент (см. §5).

---

## 5. Чего здесь НЕ делаем (явные границы)

Компонент намеренно «только фото». Смежное закрывается другими компонентами:

| Потребность | Где |
|---|---|
| Состояние **загрузки** (shimmer) | `Skeleton` (`Skeleton / Listing Card` и атомы) — не дублируем |
| **Нет фото / ошибка** (плейсхолдер с иконкой) | Состояние компонента-потребителя (карточка/галерея), не Image |
| **Favorite ♡, счётчик «1/8», тег VIDEO/360°** | `ButtonIcon Type=Overlay` / `Badge Type=Overlay` поверх — composition-rules §11 |
| **Затемнение фото для читаемости** | `Background/onPhoto` (photo-tint) на потребителе — не scrim, не здесь |
| Скрим под overlay-лейблами | связка Overlay-компонентов (§11) |

Решение зафиксировано с дизайнером 2026-06-29: «только фото, никаких состояний». Это держит атом переиспользуемым и не плодит дубли Skeleton/Overlay.

---

## 6. Применение

- **Грид листинга / категории** — `1:1` или `4:3`.
- **Wide-карточка листинга** (строка) — `16:9` или `3:2` (ср. Skeleton Listing Card Wide 343×200).
- **Галерея / hero объявления** — `4:3` / `3:2`, портретные объявления — `3:4`.
- Overlay-контролы и лейблы кладутся **поверх** инстанса Image отдельными `Type=Overlay` компонентами (§11), не встраиваются.

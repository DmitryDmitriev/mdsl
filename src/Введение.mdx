# Дизайн-система DSL

Эта документация описывает **токены** (цвета, отступы, радиусы, высоты) и **компоненты**, общие для макетов в Figma и кода. Независимо от роли — дизайнер, менеджер или разработчик — здесь можно посмотреть, как правильно использовать систему.

---

## Для кого этот Storybook

| Роль | Что смотреть |
|------|----------------|
| **Дизайнер** | Токены → Цвета, Отступы, Радиусы; компонент Button — референсы размеров и вариантов для макетов. |
| **Менеджер** | Введение (эта страница), раздел «Компоненты» — как выглядят и где применяются элементы интерфейса. |
| **Разработчик** | Все разделы; в каждой стори есть код и описание токенов для использования в коде (CSS переменные, импорты из `@/tokens`). |

---

## Структура

1. **Токены** — примитивные и семантические переменные. В UI используются только **семантические** (например `Background Primary`, `height/md`, `radius/control-md`). Подробнее в [DESIGN-TOKENS.md](/docs/DESIGN-TOKENS.md) и [COLOR-PALETTE.md](/docs/COLOR-PALETTE.md) в репозитории.

2. **Компоненты** — реализация по спекам (кнопки по [buttons-spec-figma.md](/docs/buttons-spec-figma.md)). В сториз показаны размеры (xs, sm, md, lg), варианты (Primary, Secondary, Outline, Ghost) и состояния (Default, Disabled).

---

## Правила использования

- В макетах и коде **не подставляйте** примитивные значения (Zinc/500, spacing/6) напрямую в экраны — только семантические имена.
- **Отступы:** внутри компонентов — `space`; между блоками по вертикали — `stack` (stack-sm … stack-2xl).
- **Цвета:** только роли Background, Surface, Border, Accent, Text & Icon, Decor, Brand.

Дальше — разделы **Токены** и **Компоненты** в боковой панели.
